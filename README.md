## 🌰 What This Is

Cross-chain bridges are the highest-value targets in DeFi — over **$2.85 billion** stolen across 10 major exploits. Yet most DeFi security tools focus on smart contract vulnerabilities broadly; none provide a dedicated bridge security intelligence layer.

**Bridge Security Monitor** fills this gap. It automatically:

- 🌰 Fetches live data on 100+ active cross-chain bridges (via DeFiLlama)
- 🌰 Scores each bridge's risk using a multi-factor algorithm
- 🌰 Tracks historical exploit patterns with full incident details
- 🌰 Generates weekly AI intelligence briefs (GitHub Models / gpt-4o-mini)
- 🌰 Deploys a clean dashboard to GitHub Pages — updated every week

## 🌰 Key Features

### 🔴 Risk Scoring Engine
Each bridge receives a score from 0–100 based on three weighted factors:

| Factor | Weight | Rationale |
|--------|--------|-----------|
| 💰 TVL / Volume | 0–40 pts | Larger bridges = higher-value targets |
| 💀 Exploit History | 0–40 pts | Previously compromised = higher recurrence risk |
| 🌐 Chain Surface Area | 0–20 pts | More chains = more attack surface |

**Risk levels:**
- 🔴 **CRITICAL** (≥70): Immediate attention required
- 🟠 **HIGH** (50–69): Elevated risk, monitor closely  
- 🟡 **MEDIUM** (30–49): Moderate risk
- 🟢 **LOW** (<30): Lower risk profile

### 🤖 AI Intelligence Brief
Weekly analysis generated by **GitHub Models (gpt-4o-mini)**:
- Current risk landscape summary
- Volume analysis and attacker economics
- Historical exploit pattern analysis
- Actionable recommendations

### 📊 Historical Exploit Database
Curated dataset of major bridge hacks including:
- 🌰 **Ronin/Axie Infinity** ($625M) — validator key compromise
- 🌰 **Poly Network** ($611M) — smart contract privilege escalation
- 🌰 **BNB Token Hub** ($570M) — cryptographic proof forgery
- 🌰 **Wormhole** ($320M) — signature verification bypass
- 🌰 **Nomad** ($190M) — improper input validation
- 🌰 **Multichain** ($125M) — suspected key compromise
- 🌰 **Harmony Horizon** ($100M) — multi-sig compromise
- 🌰 **Orbit Chain** ($82M) — multi-sig compromise
- And more...

## 🌰 Architecture

```
┌─────────────────────────────────────────────────────┐
│  GitHub Actions (weekly cron: Monday 00:00 UTC)     │
│                                                     │
│  1. api-call.js                                     │
│     └── GET https://bridges.llama.fi/bridges        │

         │
         ▼

```

## 🌰 Methodology Notes

### Why Bridge Security Specifically?
Bridges are uniquely risky because:
1. They hold massive reserves locked in smart contracts
2. They're complex systems with multiple trust assumptions
3. They span multiple chains — each adds attack surface
4. Compromise often means total loss (no circuit breakers)
5. Historical loss rate: $285M average per incident

### Risk Score Limitations
🌰 This tool provides **directional risk indicators**, not investment advice:
- TVL data is approximate (volume ≠ locked value)
- New bridges may have hidden vulnerabilities not captured
- Audit status is not directly factored in
- Risk scores reflect attacker *incentive*, not *capability*
